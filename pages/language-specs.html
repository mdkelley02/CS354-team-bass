Language Specification and Documents

Specification

Rust doesn't have an explicit language specification, as there is currently (as of Apr 2022) only one compiler. Since supporting portability between different compilers is often a primary motivation for writing and maintaining a language specification, this is not yet a concern for Mozilla. It does, however, have an official reference manual which is linked below.

The Rust Reference

Furthermore, since the language is under active development, there is also a manual for its unstable features:

The Unstable Book

Additionally, though they do not form an official standard, there are also the Rust RFCs:

Rust RFCs

Documentation

The following two pages form the official language documentation:

  • Rust Core Library Documentation
  • Rust Standard Library Documentation

The core library docs cover everything included by default in the prelude.
The standard library docs cover everything included with the language that must explicitly be imported.


Additional Resources

The official Rust language website also has a page where links to other miscellaneous documentation are provided by topic. The links not only cover the language, but also things like the package manager, build system, documentation generator, etc.
